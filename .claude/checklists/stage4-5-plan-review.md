# Stage 4-5: 方案设计与开发计划评审清单

## 架构对应

- [ ] 方案是否从架构蓝图拆出
- [ ] 是否覆盖所有 architecture nodes
- [ ] 模块边界是否与架构一致
- [ ] 接口设计是否与架构定义的 contract 一致

## 可执行性

- [ ] 每个模块的实现路径是否清晰
- [ ] 是否有未解决的技术不确定
- [ ] 是否有明确的数据流和状态流
- [ ] 失败/回滚/降级路径是否设计

## 可维护性

- [ ] 日志与审计是否覆盖
- [ ] 监控/告警/诊断入口是否设计
- [ ] 回滚机制是否可行
- [ ] 降级/急救模式是否考虑
- [ ] 故障恢复与人工接管入口是否设计

## 开发计划

- [ ] 是否有明确的里程碑
- [ ] 是否有排期和依赖关系
- [ ] 是否有风险点和缓解策略
- [ ] 每个功能单元的验收标准是否明确

## 乐高拆装（Rule 7）

- [ ] 方案详设粒度是否达到"子 Agent 可直接制造"
- [ ] 开发计划是否包含 Agent 分配矩阵（每个 packet 对应一个 L4 代码块）
- [ ] 每个 work packet 是否有 objective/read_scope/write_scope/acceptance/verification/rollback
- [ ] 拼装链是否定义为 L4→L0 逐级验证
- [ ] Agent 异常恢复计划是否定义
