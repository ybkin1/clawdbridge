# Stage 3: PRD 评审清单

## 完整性

- [ ] 是否覆盖用户真实目标和真实使用场景
- [ ] 是否避免只按作者假设路径设计
- [ ] 是否有明确的成功标准和验收条件
- [ ] 是否有明确的非目标（out of scope）

## 可追溯性

- [ ] 每个功能是否能追溯到 intent
- [ ] 每个功能是否有对应的验收标准
- [ ] 需求之间的依赖关系是否明确

## 可行性

- [ ] 技术方案是否可行
- [ ] 是否有未解决的技术风险
- [ ] 是否有外部依赖未确认

## 文档深度

- [ ] theme_prd 是否达到 whitepaper_like 深度
- [ ] package_prd 是否达到 detailed 深度
- [ ] 是否有模块边界、关键对象、接口/依赖关系和失败路径
- [ ] maturity_target 是否与实际深度匹配
