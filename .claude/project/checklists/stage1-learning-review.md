# Stage 1: 深度学习评审清单

## 深度检查

- [ ] 是否搜索了 ≥3 个参考项目/竞品/文档
- [ ] 是否引用了 ≥2 个深度来源（非 README/首页）
- [ ] 核心机制是否已解释（算法/架构/工作原理）
- [ ] 关键配置参数是否已列出
- [ ] 已知坑点是否已记录
- [ ] 是否有竞品对比表

## 广度检查

- [ ] 是否覆盖了 ≥2 种技术方案
- [ ] 是否考虑了替代方案
- [ ] 是否有选型决策和理由
- [ ] 是否遗漏了重要参考项目

## 架构拆解检查

- [ ] 功能拆解树是否完整（App → module → feature unit → minimal building block）
- [ ] 技术选型是否有理由和备选
- [ ] 模块交互图是否清晰
- [ ] 可维护性策略是否覆盖（目录/分层/扩展点/日志/回滚）

## 输出质量

- [ ] 学习分析报告是否包含：技术原理总结 + 关键参数 + 坑点列表 + 竞品对比
- [ ] 架构拆解清单是否包含：功能拆解树 + 技术选型 + 模块交互 + 可维护性
- [ ] 是否有独立评审（≥2 Agent 并行）
