# 用户价值检查清单（User Value Checklist）

> 本清单对应 review-gates-contract.md 的 **Value Gate**，检查产出是否满足用户真实需求。
> 所有 Standard/Complex 任务的评审中，**至少 1 个评审 Agent 必须使用本清单**。

## 1. 意图对齐

- [ ] 当前产出是否回链到 `00-user-intent.md` 中的原始诉求？
- [ ] 是否解决了用户的真实目标，而不是作者假设的目标？
- [ ] 用户要完成的关键动作，当前方案/实现是否真的支撑这些动作？
- [ ] 是否存在"看起来完成了，实际用户还得重写关键内容才能用"的情况？

## 2. 场景覆盖

- [ ] 是否覆盖了真实主路径（happy path），而不仅仅是作者方便实现的路径？
- [ ] 用户失败触发点是什么？当前方案是否仍要求用户临场补写才能避开失败？
- [ ] 是否考虑了真实运行约束？（如：数据量级、并发量、网络延迟、权限限制）
- [ ] 非功能需求是否被纳入？（性能、可用性、安全性、可维护性）

## 3. 下游可用性

- [ ] 若让下一位真实使用者"不改、不补、不脑补"直接使用当前产出，是否仍成立？
- [ ] 产出是否有可验证的 `observable_success` 定义？（不是"看起来不错"，而是"X 场景下 Y 动作能产生 Z 结果"）
- [ ] 实现/设计是否只在"理想路径"成立？真实约束下用户如何完成任务？

## 4. 交付完整性

- [ ] 文档或实现是否只描述模块/字段/流程，却没有回链到真实用户应用场景？
- [ ] 用户需求表面上被覆盖，但关键成功动作仍需下游使用者自行补写/改写/二次设计？
- [ ] 对于面向用户的交付物，是否有 `value` gate 的正式通过证据？

## 5. 价值门控裁决

- [ ] reviewer 是否能回答："目标用户拿到当前产物后，能否在不重写关键内容的前提下完成任务？"
- [ ] 如果不能，阻碍因素是什么？（设计缺陷 / 实现不完整 / 文档缺失 / 配置遗漏 / 其他）
