## 四、缺失项

### 4.1 Mobile UI 组件零测试

| 组件 | 是否有测试 | 影响 |
|------|----------|------|
| ChatScreen | ❌ | 核心页面无渲染/交互测试 |
| MessageBubble | ❌ | 5 种消息类型渲染无验证 |
| ApprovalCard | ❌ | 审批 UI 生命周期无验证 |
| InputBar | ❌ | 输入→发送流程无集成测试 |
| SessionListScreen | ❌ | 排序/状态灯/角标无验证 |
| DeviceScreen | ❌ | 配对/解绑 UI 无验证 |
| AuthScreen | ❌ | 登录表单无验证 |
| SessionCard | ❌ | 卡片渲染无验证 |

原因：Mobile 项目引入了 `react`/`react-native`/`@react-navigation` 等 UI 库用于 Expo，但 jest.config.js 配置为 `testEnvironment: 'node'`，无法运行 React Native 组件测试。**需要 `jest-expo` preset 或 `@testing-library/react-native` 才能测试 UI 组件，当前未配置。**

> **建议**：将 UI 组件测试标记为 deferred（Phase 2），或在 closeout 记录中显式声明为已知缺口。

### 4.2 跨进程集成测试缺失

当前所有测试均为**单进程 mock**测试：
- WS 测试：直接在测试中创建 `new WebSocket` 连接本地 WSServer
- Approval 测试：直接调用 `new ApprovalInterceptor()` 方法
- E2E 测试：在同一 Node 进程内启动 express 服务 + WSServer

**无** Mobile App ↔ Bridge Server ↔ Claude Code 的真实跨进程测试。

### 4.3 覆盖率不透明

Plan §11 声称 "代码覆盖率 模块级 100%"，但：
- 未配置 `jest --coverage`
- 未输出 lcov/html 报告
- "模块级" 定义模糊（按文件？按函数？按分支？）
